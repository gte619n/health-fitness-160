Health & Fitness — UI Reference

Three static mockups of the Dashboard surface across web, phone, and foldable. Use these alongside ../specs/IMPL-01-ui-design-spec.md as the visual reference for IMPL-02 implementation.

Desktop Dashboard

dashboard-desktop.html · target 1280px · primary analytical surface

Persistent 220px left nav with primary destinations, devices block, settings, and user pill. Content area shows vitals strip, body composition hero chart, two-column Blood + Today panel, recent activity feed.

Open in new tab →
Implementation notes
  • Nav active state: bg.canvas.signal pill background, no left bar. Different visual language from content section titles.
  • Section titles in content area always prefixed by a 3×11px olive vertical bar at accent.signal.
  • All numerics in JetBrains Mono with tabular figures. All caps labels in JetBrains Mono with +0.08em tracking.
  • Devices block: status dot color encodes connection state (data.good = connected, data.muted = disconnected).
  • Badge "2" on Blood: this is a notification pattern surfacing newly out-of-range markers. Use sparingly.

Phone Today

dashboard-phone.html · target 380px · capture and at-a-glance

iOS-style framing for reference (Android polish swaps the status bar conventions). 2×2 vitals grid, Today summary with calorie ring and macros, 4-tile quick log row, recent activity feed, bottom nav with four destinations.

Open in new tab →
Implementation notes
  • Bottom nav active state: 2×24px olive underline pill above the icon plus olive icon + label. Restrained but legible.
  • Quick log tiles: 80px tall, icon in accent.signal, caps mono label below. bg.surface fill.
  • For Android implementation, replace the iOS status bar with system status bar and remove the home indicator pill at the bottom.
  • Vitals labeled differently in compact form: "Resting HR" full label on phone (not abbreviated "RHR" like foldable) because the phone has fewer cards competing for width.

Foldable Dashboard (unfolded)

dashboard-foldable.html · target 760dp · book posture, vertical hinge

Icon-only collapsed left nav (64px wide). Dashboard content fills remaining canvas with the same density as desktop but slightly compressed. Subtle 1px vertical hinge crease at 7% opacity runs down the center.

Open in new tab →
Implementation notes
  • Nav active state on icon-only nav: 2px olive vertical bar on the left edge of the icon. Different treatment from desktop nav because pill background feels cramped at 38px icon size.
  • Amber dot on Blood icon replaces the "2" badge from desktop. A numeric badge would dominate the icon at this size.
  • Hinge crease is decorative only. Content spans both halves continuously. Dashboard purpose is one-glance overview, not list+detail.
  • Tabletop posture for active workout still needs a mockup. Half-fold the device, hinge horizontal: top screen for live stats, bottom for log controls. To be added.
  • Detection via WindowInfoTracker posture events. Layout collapses to phone when folded, expands to this when opened.
